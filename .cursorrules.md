# Reglas de Optimización de Prompts

## Activador (Trigger)
Siempre que el usuario inicie un mensaje con los prefijos **$P** o **$p**, la IA NO responderá directamente a la petición simple. En su lugar, deberá activar el protocolo de **Prompting de Alta Precisión**.

## El Protocolo de Conversión
Debes reestructurar y expandir la solicitud del usuario siguiendo estrictamente esta fórmula de 5 puntos:

1. **Contexto y Rol:** Define el experto ideal para la tarea (ej. Senior Backend Dev, Arquitecto de Software, experto en QA) y sitúa el problema en un entorno técnico específico.
2. **Consulta/Tarea:** Describe la acción principal de forma técnica y detallada.
3. **Especificaciones:** Incluye restricciones, lenguajes, frameworks o convenciones de código que se deduzcan del workspace actual.
4. **Criterios de Calidad:** Define qué hace que la solución sea perfecta (eficiencia $O(n)$, legibilidad, manejo de errores, tipado fuerte, etc.).
5. **Cómo debe ser la respuesta:** Estructura del output (ej. "Código completo seguido de una breve explicación de las decisiones arquitectónicas").

## Reglas de Ejecución
- **Extrapolación Inteligente:** No preguntes si puedes deducirlo. Usa el contexto de los archivos abiertos en el editor para rellenar los puntos de la fórmula.
- **Intervención Mínima:** Solo detén la ejecución para pedir información si el prompt original es tan ambiguo que impide generar una respuesta útil.
- **Optimización Silenciosa:** Transforma el prompt "detrás de escena" y entrega el resultado optimizado directamente, mencionando brevemente que se aplicó la fórmula.

## Ejemplo de Comportamiento
- **Usuario:** "$p haz un test para este componente"
- **IA:** (Aplica fórmula: Rol=Frontend Test Engineer, Tarea=Unit Testing con Jest/RTL, Criterios=100% coverage de eventos...) -> *Genera el código optimizado*.
